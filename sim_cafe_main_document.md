# [sim_cafe] main設計書[修正]

2026/04/23

------

## １，クラスの目的（概要）

#### プログラムの目的

pygletを利用してsim_cafeというカフェのシュミレーションプログラムを作ろうとしている

#### メイン制御としての役割

Main クラスがコントローラーとなりcustom_manager, seat_manager, イベントを制御する

#### 初期化の依存関係

Mainクラスが各インスタンスの受け取りを行います

#### 描写と更新

バッチを作成してon_drawで描写します。各マネージャーのupdate関数を順番に実行します。



## ２，利用モジュール一覧

- #### pyglet： 画面描写、イベント処理


- #### Map: マップ描写と統計ラベルの描写

シュミレーションゲームのUIの配置を決めている

- #### setting： シュミレーションの定数（MAP_DATA, CELL_SIZEなど）


- #### customer_manager： 顧客管理（生成・行動・削除など）


- #### seat_manager： 座席管理（座席割り当て、解放など）


- #### loguru： 状態ログ




## 3，メソッド一覧



| No    | Name       | 説明                                                         | 備考                 |
| ----- | ---------- | ------------------------------------------------------------ | -------------------- |
| **1** | `__init__` | 初期化処理。ログの初期化。ウィンドウ生成。イベント登録。Map/CustomerManager/SeatManagerの初期化、 | 外部から呼び出される |
| **2** | `on_draw`  | 画像描写イベント処理                                         | pygletイベント       |
| **3** | `update`   | 毎フレーム更新処理。CustomerManager/SeatManagerに処理を委譲  | 1/60秒ごと           |



### 3.1, コンストラクタ(`__init__`)

#### 引数：　なし

#### 処理内容：

- ログファイルの初期化、生成、キャラの状態を記録
- ウィンドウサイズを`setting`から取得
- `pyglet.window.Window`の生成（描写イベントを含む）
- `Map`・`CustomerManager`・`SeatManager`のインスタンスの生成
- `on_draw`を実行するためのイベントハンドラーをウィンドウに登録
- `pyglet.clock.schedule_interval`によりupdateを60FPSで呼び出すように設定

--------------

### 3.2, `on_draw`

#### 引数：なし

#### 処理内容

- ウィンドウをクリア
- game_screen_batch.drawで全スプライトを描写



### 3.3, update

#### 引数: 

| No   | Name      | 説明                               |
| ---- | --------- | ---------------------------------- |
| 1    | dt: float | 時間毎にupdateを更新するための時間 |



#### 処理内容: 

- `CustomerManager`/ `SeatManager`の`update`を更新
